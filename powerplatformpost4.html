Copilot Studio Project Agent

Traversing a SharePoint document library for project answers

January 20, 2025 Power Platform

Several teams were planning an EHR integration project. Timelines and budgets were being established and similar questions were coming up in weekly meetings. Project managers stored the project details in a SharePoint site library and granted access to the teams. However, the site wasn't being used. In hopes of making the information more accessible, I made a quick copilot agent to answer any questions about the project.

Knowledge

The base knowledge for the agent is the SharePoint site's document library. It holds a few word documents and excel file with project information. The settings are set to allow for generative AI to better respond to requests, but the AI's general knowledge option is turned off. I found that the general knowledge AI wasn't necessary for this project as the preference was to lean only on internal information. Side note: At the time of this post, the generative AI feature is in preview mode, so it likely wouldn't be appropriate for any externally-facing agents.

I set a couple topics to trigger for more frequently asked questions, like project cost, start date and assigned staff.

Teams integration

Authentication was required for this particular agent since the data was internal/sensitive. Microsoft appears to currently only allow for Teams integrations for Microsoft-authenticated agents. The agent appears alongside other teams and chats in the Teams application. It operates similarly to any other conversations.

Here you can see the topic set earlier spitting out a few FAQ options. The user can select them and there is a predefined response (as opposed to scanning all the documents). These run faster since the answers are already "queued up".

After it gives an answer, it will confirm that the question was answered. The conversation can still be continued even if the user says their question has been answered.

At the bottom, it indicates the cost produced is AI-generated. The project cost had a predefined answer, but there was additional cost information the agent found in the documents. Because we enabled the Generative AI option, the agent used the predefined answer and added the other cost information found. In this case, we fed it the $2 million and the maintenance and user costs it found elsewhere on the SharePoint site.
