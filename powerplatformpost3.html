Data Request Fulfillment

Collecting and centralizing data requests

November 5, 2024 Power Platform

A team was receiving frequent ad hoc data requests from regional offices via email. Requests weren't formally tracked and sometimes slipped through the cracks or were delayed. A quick power app form presented a way to standardize the requests, collect them, and put them all in one place. For this team, a SharePoint List was the best option for their backend storage.

Form

The form first requests generic contact information. Some user information (like email) is collected automatically using connectors. After the basics, the form collects information about the data request. The staff fulfilling the requests noted that frequently the staff requesting data was familiar with parts of the database schemas or knew the types of parameters needed, but did not have access to query independently. To accomodate both technical users, and potentially less technical users, we provided an option for users to add their own queries (or not). This also made it easier on internal staff if much of the query was drafted already.

Requesters are also given the option to specify the data format.

Users who requested to enter their own query are presented with an input for their query. Users who are unsure of the data location or request to provide general parameters receive the same page with slight changes to the verbage.

Users can hover over the "help" icon for recommendations on how to enter their query.

Tracking requests

The backend for the form is currently under development. It will include both a gallery for regional users to see any requests they have submitted, as well as a gallery for internal staff to see all requests.
